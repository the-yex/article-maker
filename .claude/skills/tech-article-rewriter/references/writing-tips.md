# 微信公众号技术文章写作技巧（进阶幽默版）

## 1. 彻底去除 AI 味的“人话”指南

### ❌ 拒绝“翻译腔”与“教科书腔”
- 不要写：“该算法的复杂度是 O(n)。”
- 要写：“这算法跑起来快得飞起，O(n) 复杂度稳如老狗。”

### ✅ 注入“情绪”与“槽点”
- 当技术难用时：直接吐槽“这文档写得跟天书一样”、“这 API 设计简直是反人类”。
- 当技术牛逼时：夸赞“这性能直接起飞”、“简直是后端开发者的福音”。

## 2. 图片与图解策略（核心）

### 🖼️ 原文图片保留准则
- **必须执行**：在改写过程中，原文中出现的任何架构图、流程图、数据对比图、监控截图，**必须 100% 保留在对应位置**。
- **引用格式**：保持 Markdown 的图片引用链接，确保在最终输出中不丢失。

### 💡 视觉化图解建议
- **文字图解**：如果原文没图，可以用简单的 Markdown 符号画图。
  - 例如：`请求 -> [Nginx] -> [网关] -> [微服务]`
- **类比图解**：用生活场景类比。
  - 比如解释消息队列：`“就像去吃火锅，先拿号（入队），然后在门口嗑瓜子（等待），等叫号了再进去（消费）。”`

## 3. 幽默感的来源

### 🛠️ 程序员专属比喻
- “这段代码烂得就像是在屎山上跳舞。”
- “这系统并发一高就宕机，脆得像块饼干。”
- “重构这段代码就像是在拆一颗定时炸弹。”

### 🎭 场景模拟（小剧场）
- 经常插入“我与老板的对话”、“我与产品的撕逼”、“我与 Bug 的搏斗”。

## 4. 视觉排版建议
- **短句为主**：不要出现超过 3 行的大段文字。
- **重点加粗**：把核心结论和“保命参数”加粗。
- **表情包预留**：在吐槽处可以用文字提示 `[此处建议配一个“我太难了”的表情包]`。
