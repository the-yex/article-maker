# 自检清单

文章完成后，逐项检查以下清单。任何不合格项必须立即修正后再交付。

## 开幕雷击检查

- [ ] 前 100 字是否有"雷击点"（冲突/数据/悬念/反直觉结论）？
- [ ] 开头是否直接进入核心内容（没有寒暄、没有背景铺垫）？
- [ ] 是否避免了开门见山式开头（除非干货极其突出）？
- [ ] 开头是否杜撰了虚假事件？（禁止，除非是事故类引出的思考）
- [ ] 开头是否有代入感和共鸣感？

## 完读率检查（赛马机制核心指标）

- [ ] 总字数是否在 1500-2500 字之间？（960字起步可插两条广告，超过3000字完读率断崖下跌）
- [ ] 是否存在超过 6 行的长段落？（必须拆分，段落超过6行完读率明显下降）
- [ ] 是否遵循"一句话一段，最多三句话一段"原则？
- [ ] 每 300-500 字是否有视觉变化点（表格/代码/加粗句/分割线）？
- [ ] 文章中间是否埋了 2-3 个锚点（悬念/反转/利益/共鸣）？
- [ ] 锚点位置是否合理（第一个在300-500字处，第二个在800-1000字处）？
- [ ] 是否做到了"一文一核心"（不贪多，聚焦一个知识点）？

## 结构检查

- [ ] 文章是否包含"开幕雷击 → 痛点共鸣 → 概念破冰 → 风险预警 → 技术实现 → 钩子收尾"的完整结构？
- [ ] 如果是性能类文章，是否包含性能压测对比模块？
- [ ] 文末是否有 3 条以上加粗的避坑指南？
- [ ] 文末是否有让人想转发的金句？
- [ ] 文末是否有引导评论区互动的开放性或争议性问题？

## 技术检查

- [ ] 代码是否包含错误处理和关键注释？
- [ ] 代码注释是否说明了"为什么这么写"？
- [ ] 涉及"优化""提升"的地方是否有数据支撑？
- [ ] 性能数据是否标注了来源（真实数据 / 估算值）？
- [ ] 技术方案是否有"如果不这样做会怎样"的风险分析？

## 风格检查

- [ ] 是否存在禁用词？（综上所述、值得注意的是、此外、不仅如此、在当今时代、一方面...另一方面...）
- [ ] 关键结论是否已加粗？
- [ ] 小标题是否有创意和吸引力（不是"一、二、三"式）？
- [ ] 语气是否像资深开发者聊天，而非教科书或 AI 生成？
- [ ] 是否以原创视角写作（不出现"原文""原作者"等词汇）？
- [ ] 观点是否鲜明（不骑墙、敢下结论）？

## 赛马机制优化检查

- [ ] 是否选择了蓝海领域或细分方向（避免与主流热点直接竞争）？
- [ ] 与对标文章相比，是否做到了差异化（人无我有，人有我优）？
- [ ] 标题是否考虑了转发场景（无心理负担，转发后显得有见识）？
- [ ] 是否包含可截图转发的内容（表格/金句/Checklist）？
- [ ] 互动引导是否能有效提升评论数（争议性或选择性问题）？

## 传播力检查

- [ ] 标题是否控制在 25 字以内，关键信息前置？
- [ ] 标题是否包含具体数字？
- [ ] 标题转发到朋友圈是否会让转发者有心理负担？
- [ ] 文章是否包含可被截图转发的内容（表格/金句/Checklist）？
- [ ] 避坑指南是否具有"转发给同事看"的实用价值？
- [ ] 文章是否有社交货币价值（转发后显得有见识/有态度）？

## 资产检查

- [ ] 原文中的所有图片 URL 是否 100% 保留？
- [ ] 图片是否放在文章中的对应位置？
- [ ] 架构图、流程图、数据对比图是否完整？

## 修正优先级

发现问题时，按以下优先级修正：

1. **开幕雷击与完读率**（最高，直接影响赛马机制中的算法推荐）
2. **结构与逻辑错误**
3. **技术深度与准确性**
4. **传播力与互动设计**（影响点赞率、转发率、评论数）
5. **风格与语言规范**
6. **排版与视觉效果**（最低）
