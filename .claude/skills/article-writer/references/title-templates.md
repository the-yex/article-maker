# 爆款标题模板

## 标题底层逻辑

> 文章阅读量 = (粉丝基数 × 打开率) + (转发人数 × 朋友圈人数 × 朋友圈打开率)

标题决定打开率，也影响转发率。好标题必须同时满足两个条件：**让人想点开** + **让人敢转发**。

## 标题基本规范

| 规范 | 要求 |
|:-----|:-----|
| 字数 | 控制在 15 字以内，信息关键词前置 |
| 3 秒法则 | 读者 3 秒内决定是否点击，核心信息必须在前半段 |
| 转发友好 | 标题转发到朋友圈不会让转发者有心理负担（避免过度标题党） |
| 包含数字 | 数字比文字更抓眼球，优先使用具体数字 |

## 六种爆款标题类型

每次生成 **5 个候选标题**，覆盖至少 3 种类型。

### 类型一：冲突反转（高点击率）

制造认知冲突，用亲身经历打脸"大家都信"的观点。

- `我以为 Redis 永远不会挂，直到凌晨 3 点被叫醒`
- `别再盲目上微服务了！这个坑让我们回退到单体`
- `都说 Go 性能好，直到我写出了比 Python 还慢的代码`

**公式**：[大众认知] + [反转事实/亲身打脸]

### 类型二：极限场景（权威感）

用大数字和极端场景建立专业感，让读者觉得"这个人有料"。

- `日均 10 亿请求，我们是怎么把延迟压到 5ms 的`
- `单机百万连接：一次 Netty 调优的完整记录`
- `从 30 秒到 300 毫秒：一条慢 SQL 的优化全过程`

**公式**：[大数字量级] + [具体技术手段] + [量化结果]

### 类型三：经验自嘲（共鸣感）

用踩坑经历引发共鸣，让读者觉得"这说的就是我"。

- `踩了 3 年的坑，我总结出 5 条 Docker 避坑指南`
- `如果早知道这个配置，我就不用半夜爬起来了`
- `一次线上事故，让我彻底搞懂了分布式锁`

**公式**：[时间/次数/事故] + [踩坑经历] + [总结/教训]

### 类型四：利益直击（获得感）

直接告诉读者"看完能得到什么"，降低决策成本。

- `学会这招，SQL 查询速度提升 100 倍`
- `这份 JVM 调优清单，建议每个 Java 开发收藏`
- `面试必考的 8 个 Redis 问题，看完直接能答`

**公式**：[动作指令] + [具体收益/数字]

### 类型五：大厂背书（流量加持）

技术公众号中，大厂名字自带流量。

- `字节跳动内部的 Go 代码规范，今天终于公开了`
- `阿里 P8 的代码审查清单，每一条都是血泪教训`
- `美团面试官问我：线程池参数怎么配？我这样回答`

**公式**：[大厂/职级] + [具体技术内容] + [稀缺感/故事感]

### 类型六：悬念好奇（信息缺口）

制造信息缺口，让读者"不点开就难受"。

- `这个 Bug 藏了 2 年，排查过程让我大开眼界`
- `为什么你的接口总是超时？答案可能和你想的不一样`
- `同事写的代码，差点让公司赔了 200 万`

**公式**：[悬念/异常现象] + [暗示有意外答案]

## 升维词替换表

普通词汇替换为更有冲击力的表达，提升标题吸引力：

| 普通词 | 升维词 |
|:-------|:-------|
| 方法 | 狠招 / 绝招 / 杀手锏 |
| 技巧 | 骚操作 / 黑科技 |
| 总结 | 血泪教训 / 踩坑实录 |
| 推荐 | 私藏 / 压箱底 |
| 重要 | 救命级 / 保命指南 |
| 好用 | 真香 / 神器 |
| 问题 | 天坑 / 暗雷 |
| 学会 | 吃透 / 拿下 |
| 优化 | 压榨 / 榨干 |
| 程序员 | 打工人 / 码农 |

## 标题自检

生成标题后，用以下 5 个问题自检：

1. 读者 3 秒内能看懂标题在讲什么吗？
2. 标题是否包含具体数字或量化信息？
3. 读者转发到朋友圈会不会觉得尴尬？
4. 标题是否触发了好奇心、共鸣感或获得感中的至少一种？
5. 标题是否和同类文章有差异化？（避免千篇一律）
